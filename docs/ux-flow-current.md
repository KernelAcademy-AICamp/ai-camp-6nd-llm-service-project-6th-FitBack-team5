# FitBack — 현재 전체 시스템 UX Flow (AS-IS)

문서 버전: v1.0 (현재 코드 기준 검토)
작성일: 2026-06-18
기준: `myeongjin` 브랜치 현재 상태 (aera·yewon 머지 + 회원권 2종화·활용도 홈 통합·체크인 검증 반영).
근거: 실제 라우트/컴포넌트 확인. 추정·미구현은 "⚠"로 표기.

## 0. 한눈에 (관통 흐름)
```
인증/온보딩 → 홈(활용도+코치 허브) → [회원권 출석=체크인 / 식단 / 운동] 실행 → 기록 → 홈 갱신
                         └ 코치 챗봇(CoachChat) = 홈·운동에서 진입하는 단일 허브
```

## 1. 내비게이션 구조

### 1.1 하단 탭 — 3탭
`홈(/)` · `식단(/diet)` · `운동(/workout)`
- 웹=expo-router/ui Tabs, 네이티브=NativeTabs. 운동 탭은 하위(coach·session·complete) 어디서든 누르면 운동 홈으로 리셋.

### 1.2 비탭 라우트 (홈에서 진입)
- `/membership` — 회원권 목록·추가·상세 (홈 '전체보기'·알림 배너·회원권 카드에서 이동)
- `/my` — 마이 (※ 홈 상단 ☰는 **인라인 드로어**로 열림 — `/my` 라우트와 기능 중복 상태, §7)

### 1.3 글로벌 상단바 (현재 홈에 구현)
`☰ 마이(드로어)` · `📅 일정 캘린더(모달)` · `🔔 알림(모달, 만료 시 배지)`
- ⚠ 식단·운동 탭에는 아직 미적용(각자 헤더) — "글로벌"은 홈 한정.

## 2. 인증 / 온보딩
```
앱 시작 → useAuthBootstrap
  ├ 비로그인 → 로그인/회원가입(이메일+비번)  ⚠ 비번 재설정 없음
  └ 로그인 → onboarded=false → 온보딩 5단계
        (정보 → 회원권+센터[OCR/검색/GPS] → 형태[기간권/인세권] → 상세[기간권=주당목표 / 인세권=횟수] → 운동목표)
     → onboarded=true → 홈(3탭) 진입
```

## 3. 탭별 Flow

### 3-1. 홈 (활용도 + 코치 허브)  — `index.tsx`
```
상단바: ☰ 마이 · 📅 캘린더 · 🔔 알림(배지)
 ├ 알림 배너: 만료 임박(D-7) 회원권 → 탭 시 /membership
 ├ AI 코치 말풍선(coach 1줄) → 탭 시 코치 챗봇(CoachChat) 모달
 ├ 메인 활용도 카드:
 │   · "오늘 체크인으로 +₩N UP"(회당 회수) · 누적 회수(상승) · 활용도 바(목표75%, 25/50/75 마커)
 │   · 3칸: 회원권 수/회수 · 목표까지 남은 · 원금
 │   · [회원권 출석] → 체크인 플로우(모달)
 └ 등록 회원권 캐러셀(MembershipStatsCard) + [전체보기] → /membership
```
- 데이터: `useMemberships`(2종) + `portfolio`(목표 회수) + `coach`.

### 3-2. 식단 — `diet.tsx` (yewon)
```
날짜 네비 → 목표 대비 매크로 카드 → 끼니 기록(2×2)
 입력: 사진(Claude Vision) · 텍스트 · 식약처 검색 · 즐겨찾기
 결과: 점수·기여도 · 트레이너 피드백 · 다음 끼니 추천
```
- 현재 = 기록·입력 중심(추천은 결과 이후). → 코치-중심 전환 대상(§6).

### 3-3. 운동 — `workout/` (aera)
```
운동 홈(index) → 조건 입력/코치 질문(coach 챗봇) → AI 루틴 생성
 → 세션(session: TTS 음성코칭·타이머·세트) → 완료(complete: 난이도/통증 → AI 피드백 → workout_logs)
```
- AI 루틴/피드백 = 서버(`workout-ai`) 프록시. ⚠ 홈트 기록(workout_logs)은 센터 방문(visits)과 분리 — ROI 미반영.

## 4. 공통 모달 / 허브

### 4-1. 체크인 (CheckInFlow, 5단계 + 시나리오 C)
```
준비("오늘 +₩N 되찾아요" 미리보기) → 출발(길안내: 자동차/도보·대중교통/날씨)
→ 가는중(실시간 위치·격려) → 도착(지오펜스 100m 검증)
     ├ 검증(VERIFIED): "+₩N 되찾았어요" + 단계 마커, visits(method=geofence, recovered=perVisit)
     └ 벗어남/GPS X: 자기신고(FALLBACK) — 'GPS 미검증·회수 미반영'
→ 이동기록(형태별 exercise_records) → 홈 복귀
```

### 4-2. 코치 챗봇 (CoachChat) — 단일 허브
- 진입: 홈 코치 말풍선, 운동 탭 coach 라우트.
- 운동·식단 질문 인라인 처리(루틴 생성/식단 추천), AI 코치 피드백.

### 4-3. 일정 캘린더 (MonthCalendar 모달)
- 월 이동 + 일별 방문/운동/식단 + 날짜 선택 기록 리스트(식단 딥링크) + 권장 페이스.

### 4-4. 알림 (모달)
- 만료 D-7 회원권 노출, 없으면 "새 알림 없음". ⚠ 푸시 없음(홈 배너·🔔로 대체).

### 4-5. 마이 (☰ 드로어)
- 프로필·키/몸무게·About·로그아웃.

## 5. 핵심 데이터/검증 규칙
- **회원권 2종**: `period`(기간권: 주당 목표 환산) / `session`(인세권: 횟수). 회당 가치·목표(원금75%)·회수·단계 = `portfolio`.
- **검증 방문**: 지오펜스 100m = `verified`(회수 반영), 자기신고 = `unverified`(회수 0). `visits.method/verify_status/recovered_amount` 저장.
- **AI**: coach·coach-chat·food-search·workout-ai 전부 Supabase 시크릿(`ANTHROPIC_API_KEY`) 서버 프록시. 클라 키 없음.

## 7. 알려진 갭 / 검토 사항
1. **글로벌 상단바가 홈에만** — 식단·운동 탭 공통 적용은 팀원 파일 수정(충돌 위험) 필요.
2. **/my 라우트 ↔ ☰ 드로어 중복** — 하나로 정리 필요(드로어 권장).
3. **"회수=검증 방문만"** 화면 반영 시 `useMemberships` 방문 카운트를 verified-only로 분리해야 함(홈 영향) — 별도 검토.
4. **로그인 비번 재설정 없음** — 출시 전 필요.
5. **푸시 알림 미구현** — 홈 배너/🔔로 대체 중.
6. **공유 DB 스키마 변경**(회원권 2종·visits 검증 컬럼) 적용됨 — 옛 3종 코드 브랜치와 충돌, 팀 공지·머지 조율 필요.
