# FitBack — 홈 다이나믹 개선 방향 (현 구조 유지)

문서 버전: v0.1
작성일: 2026-06-19
근거: 사용자 피드백("홈이 정적·죽은 느낌, AI 선생님 과노출, 분석·통계로 다이나믹하게, 2상태=운동/회원권, 회원권 개수 먼저+카드 축소+빈 상태") + 현재 `index.tsx` 구조 + 보유 데이터 확인.

---

## 0. 원칙
- **전면 개편 X** — 블록 *재배치 + 비주얼/모션 강화*, 기존 컴포넌트 재사용, **신규 백엔드 0**.
- 홈을 **2개 상태**로 묶는다: ① **내 운동 상태**(행동·분석) ② **회원권 상태**(ROI·돈).
- AI 코치는 단독 노출 제거 → **"분석 해설자"로 톤다운**(카드 안 한 줄로 흡수).

## 1. 보유 데이터(추가 백엔드 0)
- `useHomeActivity`: `weekDays`(이번 주 7일)·`weekVisits`·`weekWorkouts`·`streakWeeks`
- `useVisitPattern`: `byWeekday`·`byTimeBucket` + `describeWeekday()`
- `useMonthlyStats`: 월 방문수·`byPart`(부위별)·회원권별
- `portfolio`: 회수액·활용도%·오늘 +₩N·단계

## 2. 블록 순서 재배치
```
헤더 (유지)
알림 배너 (유지)
① 내 운동 상태 카드  ← 코치 말풍선(단독) 대체
② 회원권 상태
   ├ 요약 헤더 "회원권 N개 · 활성 N · 임박 N"
   ├ ROI 미니(회수·활용도, 카운트업)
   └ 축소된 회원권 리스트 + 전체보기
   (0개면 → 빈 상태)
```

## 3. ① 내 운동 상태 카드 (다이나믹 핵심)
- 주간 7일 막대/점(`weekDays`) — 채워지는 모션
- 이번 주 목표 게이지(`weekVisits` / 개인화 권장 횟수)
- 연속 스트릭 불꽃(`streakWeeks`) — 카운트업
- 패턴 인사이트 1줄(`describeWeekday`/`byPart`) — "분석되는" 느낌
- 코치 한마디 1줄(`coach`) — 분석 해설(말풍선 단독 → 카드 캡션 흡수). 탭 → 챗봇 유지

## 4. ② 회원권 상태 (개수 먼저·부피 축소·빈 상태)
- **요약 헤더**: `회원권 N개 · 활성 N · 임박 N` (개수 먼저)
- **ROI 미니**: 누적 회수(카운트업) + 활용도 링/바 + "오늘 +₩N"
- **카드 부피 축소**: 큰 캐러셀 → 한 줄 리스트(이름·D-day·미니 활용도바), 상세는 전체보기
- **빈 상태(0개)**: "회원권 등록하고 본전 찾기 시작" + [등록] + 한 줄 설명

## 5. 코치 톤다운
- 상단 단독 말풍선 제거 → ① 카드 하단 "코치 한마디"로 통합.
- 챗봇 전면 대안은 현 구조 유지 원칙과 어긋나 비권장(진입 버튼만 강조).

## 6. "죽은 느낌" 제거 — 연출 레이어
- 숫자 카운트업, 진행바·링 채우기, 주간 막대 grow, 단계 도달 강조.
- 로딩 시 스켈레톤(정적/빈 화면 방지). (Animated/Reanimated)

## 7. 유지 / 안 함
- 유지: 헤더·알림·체크인 플로우·`회원권 출석` CTA·캐러셀 데이터.
- 안 함: 라우트/탭 구조 변경, 신규 백엔드, 전면 리디자인.

## 8. 우선순위
- **P0**: 운동 상태 카드(주간그래프+스트릭+목표) · 코치 통합 · 회원권 개수헤더/카드축소/빈상태
- **P1**: 패턴 인사이트 1줄 · 카운트업/모션
- **P2**: 지난주 대비 증감(↑↓) — 소량 계산 추가 필요(현재 미보유)
