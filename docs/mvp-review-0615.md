# FitBack MVP 기능 검토 & 개선 우선순위 (26.06.15 ~ 06.19)

> **v2 — 팀 공유 우선순위/OKR v3(2026-06-15) 반영본.** 코드 인벤토리(사실) 기반 + 팀 확정 결정 정렬.
> 핵심 3대 기능 = **회원권 · 식단 · 운동(홈트)**. 용어: "본전" → **"회원권 활용도"**로 통일.
> 페르소나: 주타겟 여성 20–35, 세컨드 남성. 이번 스프린트는 **무료(자유이용)권 사용자 중심**.
> 우선순위: **P0/P1/P2 = MVP 필수도 × painpoint** + 노력(S/M/L) 별도.

---

## 0. 우선순위 / 노력 정의

- **P0** — 핵심 길(가입→대시보드→체크인→기록→활용도 확인) 또는 **출시 차단(보안/신뢰)**. = 1차 배포 게이트.
- **P1** — 핵심 보조: 행동 전환(코치·배너), 꾸준함 노출, 필수 화면 최소 완성도. 배포 후 '개선 작업'.
- **P2** — 핵심 3대 중 **신규 없이 정확도만** 손볼 영역(식단·홈트) + 정리성 작업.
- 노력: **S** ≤0.5일 · **M** ~1일 · **L** ~2일+ (1인 개발 기준)

> 참고: 식단·운동은 **핵심 3대**지만 *완료* 상태라 이번 스프린트엔 신규 기능을 넣지 않고 **정확도/정리**만 한다(팀 결정 5). ROI 가치 사슬의 *중심*은 회원권이다.

## 1. 페르소나 & 핵심 여정

| | 주 페르소나 A | 세컨드 B |
|---|---|---|
| 정의 | 여성 20–35, 헬스장 끊었지만 잘 못 감. "돈 아까운데 못 가는" 죄책감 | 남성, 데이터·효율 중심 |
| 회원권 형태 | **무료(자유이용)권을 가장 많이 씀** | 다양 |
| painpoint | ① 회원권이 새는지 **체감 안 됨** ② 갈 동기·계기 부족 ③ 가도 기록 안 남아 성취감 X | 통계·진척의 정량 피드백 |
| FitBack의 약속 | **회원권 활용도를 보여주고**, 다음 행동을 제안하고, 가면 회수로 보상 | 데이터 밀도 높은 활용도 뷰 |

**핵심 전환 여정(이번 스프린트가 반드시 매끄럽게 할 길):**
`온보딩(회원권 등록)` → `대시보드/회원권(활용도·손실 인지)` → `AI 코치(행동 제안)` → `오늘 체크인(검증된 방문)` → `기록(꾸준함)` → `대시보드(활용도 확인)`

> ⚠️ 병목 = **"체크인 전환"**과 **"활용도의 체감"**. 무료권 사용자에선 활용도가 *금액*이 아니라 **습관·페이스**로 체감된다(팀 결정 1).

## 4. OKR (스프린트 06.15 ~ 06.28)

**Objective: 무료 회원권 사용자 중심으로 핵심 길이 막힘없이 돌고, 보완·필수 화면을 갖춘 "공개 가능한 MVP"를 완성한다.**

| KR | 무엇을 | 통과 기준 |
|---|---|---|
| KR1 | 핵심 길(가입→대시보드→체크인→기록→활용도)이 막힘없이 | **시나리오 4종**(정상/회원권0/무료권만/만료) 에러 없이 통과(체크리스트 1회씩) |
| KR2 | 무료 회원권 활용도(습관·페이스)가 제대로 | 방문 **0회 포함 모든 상태**에서 빈칸·음수·이상 숫자 **0건**, 0회 안내 문구 노출(상태별 캡처) |
| KR3 | 운동 AI 키가 앱에 안 남나 | 빌드 클라이언트에서 `EXPO_PUBLIC_ANTHROPIC_API_KEY` 검색 **0건** |
| KR4 | 출시 전 필수 3종 | 비번 재설정 동작 · **실기기 1대** GPS 체크인 성공 · 미완 버튼 숨김 |

> KR은 "공개 가능한 완성도"(이번 스프린트에서 측정 가능)를 본다. 실제 "갈 만하다" 체감(체크인 전환율 등)은 베타 측정 목표. 4개 KR 모두 통과하려면 P0가 먼저 들어가야 하므로, **막히면 P1을 먼저 미루는 컷**을 전제로 한다.

## 5. 확정된 제품 결정 (팀 v3 반영 — 기존 §5 미결 → 결정)

1. **무료(자유이용)권 활용도 = "습관·페이스"**
   - **1순위(히어로): "이번 주 N/목표회 · 연속 N주"** (페이스 게이지). 0회 → "이번 주 첫 운동 기록해보세요".
   - **보조 줄: "이번 달 회당 N원↓"** (첫 방문 후에만 표시).
   - **시각: 활용도 % 게이지.**
   - 이유: 무료권은 스트릭이 금액보다 자연스러운 동기. 회당비용은 0회에서 깨지므로 보조로만.
2. **체크인 실패 시 "운동기록 출석 보정" = 유지** — 목적은 무조건 출석을 남기기 위함. 단 활용도 신뢰를 위해 **자기신고 방문 ↔ 검증 방문 구분 표기(약하게).**
3. **MY 코치(대화형) = 진입 버튼 숨김** — 보류 기능이라 안 보이게. 코드 보존, 나중에 한 곳만 연결하면 재개.
4. **알림(푸시) = 이번 MVP 미포함** → 대신 **홈 경고 배너 강화**로 대체.
5. **식단·홈트 = 신규 기능 없음, 정확도만 개선** — 특히 홈트는 **운동 동작 이미지 추가** 예정.

### 6-2. 항목별 "핵심 AI로 적정한가"

- **로그인** — AI 불필요. 적정.
- **온보딩/회원권(영수증 OCR)** — 입력 마찰 감소 방향은 적정, 단 **미동작 스텁**. → **식단 Vision(Claude) 인프라 재사용으로 "영수증 사진→회원권 필드" 파싱** 권고(웹/네이티브 공통, 서버 키).
- **대시보드(이번 주 추천)** — **앱의 핵심 AI로 가장 적정.** 짧은 3필드 출력엔 **Haiku 검토**(비용).
- **MY 코치 채팅** — 현 단계 핵심 AI로 부적정(범용 챗봇, ROI 초점 약함). **버튼 숨김(결정 3)이 타당.**
- **회원권** — 활용도 신뢰는 결정론적 산식에서 나오며 **AI가 핵심일 필요 없음(적정).** LLM에 숫자를 안 맡긴 현 설계가 옳음.
- **식단** — 음식→영양 추정은 LLM/Vision 강점이라 적정, 모델 티어링 합리적. 이번 주는 **정확도만**.
- **운동** — 기능은 적정하나 **클라 키(보안)** → Edge Fn 이관 필수(P0-1). 완료 피드백은 짧으므로 **Haiku로 충분.**

### 6-3. 종합 진단

1. 🔴 **ROI 정렬 역전** — 핵심 AI(코치 1종)는 빈약, AI 투자는 비핵심 작업(식단4·운동2)에 집중.
2. 🔴 **거버넌스 불일치** — 식단·코치는 서버 프록시, **운동만 클라 키** → 통일 필수(P0-1, KR3).
3. 🟡 **모델 티어링** — 짧은 출력(코치 카드·운동 피드백)은 **Haiku 통일 검토**, 분석/생성만 Sonnet.
4. 🟡 **미동작 입력 AI** — 영수증 OCR 스텁 → 식단 Vision 재사용 권고.
5. 💡 **누락된 핵심 AI(후보)** — 손실 예측, 개인화 리마인더, **무료권 전용 코치**(결정 1과 연결).
