# FitBack — "코치·추천 중심" 전환 UX Flow 방향서

문서 버전: v0.1 (방향 제안 / 구현 전)
작성일: 2026-06-18
전제: **현재 디자인·블록 UI·기존 기능 유지**. 기록 중심 → 코치·추천 중심으로 **무게중심만 이동**.

---

## 0. 핵심 전환 원칙

> **각 탭 최상단 = "지금 뭘 할까(추천 1개 + 실행 버튼)" 블록**을 먼저 둔다. 기록·통계는 그 아래로 내린다.
> 무게중심: *"내가 입력한다"* → *"코치가 제안 → 한 번 탭으로 실행한다"*.

- 모든 추천 블록은 **블록 안에 실행 버튼**을 둔다(시안의 블록형 UI 유지). 예: `[회원권 출석]` `[루틴 시작]` `[이걸로 기록]`.
- 추천이 비거나 로딩 중이면 "기록해보세요" 식 **시작 유도**(기존 톤 유지, 빈 화면 금지).
- 블록의 **"더 물어보기"는 전부 코치 챗봇(CoachChat)으로 연결**(이미 존재 — 단일 허브).

---

## 1. 공통 패턴 (탭 3개 동일 골격)

```
[① 코치 추천 블록]   ← 탭 진입 시 가장 먼저 (탭 맥락에 특화)
[② 기존 핵심 지표/기록]   ← 현재 화면 그대로, 한 칸 아래로
[③ 기존 보조 블록]
```

- ①은 **신규 컴포넌트 최소** — 기존 코치 말풍선/카드 스타일 재사용.

---

## 2. 탭별 방향

### 2-1. 홈 (종합 코치 허브)
- **현재**: 활용도 헤더가 최상단, AI 코치 말풍선이 근처.
- **방향**: 말풍선을 **"오늘의 추천 카드"**로 승격 — *오늘 가장 중요한 행동 1개* + 실행 버튼(`[회원권 출석]` / `[홈트 5분]` / `[식단 추천]`). 활용도 헤더·회원권 카드·하단 기능 블록은 **그대로 아래로**.
- **데이터**: 회원권/방문 종합(`coach`). 카드 탭 → **코치 챗봇** 진입.

### 2-2. 식단
- **현재**: 매크로 카드·끼니 기록·AI 분석이 중심(기록이 먼저).
- **방향**: 최상단에 **"오늘 식단 추천" 블록** — 부족 영양소 기반 *다음 끼니 1개 추천* + `[이걸로 기록]` 버튼(기존 recommend 재배치). 그 아래 기존 매크로·끼니 기록 유지.
- **요지**: "뭘 먹지? → 추천 → 한 번에 기록"이 먼저, 수동 입력은 다음.
- **데이터**: `food-search`(recommend).

### 2-3. 운동
- **현재**: 운동 홈/주간 기록 + AI 루틴 생성(coach 라우트).
- **방향**: 최상단에 **"오늘 운동 추천" 블록** — *오늘 루틴 1개 제안*(컨디션/부위 기반) + `[바로 시작]` / `[루틴 만들기]` 버튼. 그 아래 주간 기록·세션 유지.
- **요지**: 추천 루틴에서 **바로 세션으로** 연결. 기존 운동 코치 챗봇을 블록에서 진입.
- **데이터**: `workout-ai`(루틴).

---

## 3. 전체 흐름 (코치가 관통)

```
홈(종합 추천·허브) ─┬─→ 식단 탭(식단 추천 → 기록)
                    ├─→ 운동 탭(루틴 추천 → 세션)
                    └─→ 회원권 출석(체크인)

   각 탭 추천 블록의 "더 물어보기" → 코치 챗봇(CoachChat) 공통 진입
   실행/기록 완료 → 홈 추천 갱신(다음 행동 제안)
```

- 코치 챗봇은 **단일 허브**(홈 + 각 탭 추천 블록에서 진입) → "추천 중심" 일관성 확보.

---

## 4. 검토 포인트 (진행 전 확인)

1. **식단·운동 탭은 팀원(yewon/aera) 파일** → 최상단 추천 블록 추가 = 해당 화면 수정(충돌 위험). 두 탭을 지금 직접 수정할지, 아니면 **홈 추천 카드부터(내 파일)** 먼저 하고 식단·운동은 팀 조율 후 할지.
2. **추천 데이터 소스**: 식단=`food-search`(recommend), 운동=`workout-ai`(루틴), 홈=`coach` — 전부 존재(Anthropic 크레딧 정상). 추가 백엔드 없이 **배치만** 바꾸면 됨.
3. **디자인**: 추천 블록을 기존 코치 말풍선/카드 스타일 재사용으로 갈지(신규 컴포넌트 최소).

**권장 우선순위**: ① 홈 추천 카드 승격(내 파일, 충돌 0) → ② 식단·운동 추천 블록(팀 조율 후).

---

## 5. 범위/원칙 메모
- 기존 기능은 **삭제하지 않음**(아래로 재배치만).
- 신규 화면 최소, 신규 백엔드 없음.
- 카피·톤은 `design-system.md` / 코치 Voice&Tone 준수(평가 X·응원·본전 프레이밍, 이모지 금지).
