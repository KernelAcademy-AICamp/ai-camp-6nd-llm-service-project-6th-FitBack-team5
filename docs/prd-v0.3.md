# 01. Background

### 현실의 문제

많은 사람이 헬스장·PT·필라테스·요가 등 운동 **회원권을 구매**한다. 하지만 결제 이후:

```
❌ 회원권을 얼마나 효율적으로 쓰는지 알기 어렵다
❌ 운동을 꾸준히 다니지 못한다 (일정 불규칙·동기 저하)
❌ "내 운동이 효과가 있는지" 결과가 보이지 않는다
❌ 결국 기간이 종료되며 투자금이 회수되지 않는다
```

### VOC가 말해준 것 (n=21, 구매자 14) — 가설 재정렬

기존 제품 가설은 "회원권이 **소멸/마감**되는 불안"이었으나, VOC는 이를 **거의 기각**했다.

| 가설 | 데이터 | 판정 |
|---|---|---|
| 마감·예약 깜빡함이 핵심 장벽 | 깜빡함 = 1명 | ❌ 거의 기각 |
| 자주 못 가서 **돈이 아깝다** | 10/14 | ✅ 지지 (단, 체념에 가까움) |
| 미방문 1위 이유 = **동기부여 어려움(결과가 안 보임)** | 8/14 | ✅ **핵심 장벽** |

> **결론**: 진짜 장벽은 "마감"이 아니라 **"동기·결과 부재"**. 사용자가 직접 1순위로 꼽은 기능은 **운동 효과 분석 + 일정 최적화**다.
> 한편 **회당비용/활용도**를 스스로 1순위 기능으로 꼽은 사람은 21명 중 1명뿐이지만, **보여주면 76%가 "동기로 작동, 확인하고 싶다"**고 답했다.

> **시사점**: 회당비용·회원권 활용도는 *사용자가 검색하는 "기능"이 아니라, 동기로 작동하는 "프레이밍·연출 레이어"*다. → **히어로는 '효과가 보이는 분석', ROI(회원권 활용도)는 이를 떠받치는 동기 장치**로 재배치한다. 블랭킷 '마감 손해' 정서·알림 카피는 근거가 약하므로 **응원·활용도 프레이밍**으로 전환하되, 위협 표현은 형태별로 정직하게(세션권=소멸·기간권=정체) 쓴다.

# 02. Problem Statement

기존 서비스는 문제를 따로 푼다 — 운동 앱(기록) / 식단 앱(칼로리) / 회원권 앱(예약·결제). 그러나 사용자의 질문은 하나다.

> **"내가 운동에 투자한 돈과 시간이, 지금 제대로 효과를 내고 있는가?"**

현재 시장에 부족한 것:

```
✗ 회원권 활용도(투자 회수)를
✗ 운동 효과 분석(개인화)과
✗ 식단·습관 코칭으로 연결하면서
✗ "다음에 뭘 할지"까지 제안하는 AI 페이스메이커
```

**FitBack의 해결책 (VOC 반영)**

```
✅ 결과를 먼저 보여준다 — 주간 추이·목표 달성도·부위 분석(동기 #1 장벽 정면 대응)
✅ 회원권을 활용도(회수)의 '원금'으로 — 방문 = 회수, 회당비용 하락을 연출 레이어로
✅ 검증된 방문만 회수에 반영 — 지오펜스 100m(신뢰)
✅ 코치가 다음 행동을 제안 — 각 탭 최상단 추천 + 코치 챗봇 허브
✅ 응원·활용도 톤 — 죄책감/공포 대신 통장·재테크 + 페이스메이커 비유 (위협은 형태별 사실로: 세션권 소멸·기간권 정체)
```

# 03. Target User

### 🎯 Primary — 여성 직장인 (25~39세)

**한 줄**: 회원권은 끊지만 불규칙한 일정과 동기 저하로 끝까지 활용하지 못하는 직장인.

- 헬스·필라테스·요가 보유, 한 센터에 정착, 환경·트레이너 중시
- **Pain**: ①가도 효과가 보이지 않아 동기가 떨어진다 ②일정이 불규칙해 자꾸 미룬다 ③회원권을 다 못 쓸 것 같은 막연한 불안
- **JTBD**: "내 운동이 효과 있는지 확인하면서, 최소한의 노력으로 꾸준히 다니고 싶어."

### 🔬 확장 가설 — 남성 직장인 (30~39세) · **미검증, Phase 2 검증 대상**

**현재 타깃 아님.** VOC 표본에서 남성·PT(세션권) 세그먼트는 검증이 약하고(VOC §4), 제품 핵심 가치는 여성 주타깃 특성에 맞춰져 있다. 동급 Secondary로 두면 베타 포커스가 흐려지므로, **여성 25–39 단일 주타깃**으로 집중하고 남성은 **데이터 축적 후 검증할 확장 가설**로 보류한다.

- 가설: 결과·데이터 중시 → 효과 분석·활용도 시각화가 통할 수 있음.
- 검증 방법(Phase 2): 남성·세션권 표본 확보 후 A/B, 효과분석 열람률·코치 추천 실행률 비교.

> **VOC 표본 해석 기준** — 표본의 여성 62%·20~24세 비중은 *설문 배포 대상 자체가 그 연령대에 집중*된 결과다(모집단 편향이 아닌 수집 편향). 따라서 VOC 분석은 **주타깃(여성 25~39) 렌즈로 읽는다**: 20~24·미구매 응답은 보조 참고로 두고, 구매자(n=14)·주타깃에 해당하는 신호(동기·결과 가시성·일정 불규칙)에 가중한다. 주타깃 표본 보강은 §10.

# 04. Product Vision

**투자한 비용의 회수와 운동 효과를 눈에 보이게 만들고, AI 페이스메이커가 다음 한 걸음을 제안해 끝까지 동반한다.**

```
【회원권(돈)】  회원권 활용도 모델 — 회당가치·목표 75% 회수·단계 마라톤
【운동(행동)】  주간 추이·스트릭·부위 분석 + 검증 체크인 + AI 루틴·음성 세션
【식단(습관)】  목표 대비 달성 + 다음 끼니 추천(이미지/검색)
【AI 코치】     데이터 해석 + 다음 행동 제안(탭별 추천 + 챗봇 허브)
```

페이스메이커 컨셉: *통장 잔고를 채우듯 회원권 활용도를 쌓고, 코치가 페이스를 잡아준다.* 평가하지 않고 데이터를 보여주며 다음 행동을 제안한다.

# 05. Core Features

## 아키텍처: 공통 모듈 + 3개 도메인 + AI 코치(관통)

```
        【공통】 사용자·회원권(2종)
                 ↓
   ┌─────────────┼─────────────┐
 【홈/회원권】  【식단】      【운동】     ← 3 탭
  ROI·효과분석   AI식단코칭    AI루틴·세션
                 ↓
        【AI 코치】 탭별 추천 블록 + 코치 챗봇(단일 허브)
```

---

## 🔧 모듈 1 — 공통 (사용자·회원권)

### 1-1. 프로필·온보딩 (구현됨)
정보 → 회원권+센터 등록 → 형태 선택(기간권/세션권) → 상세(기간권=주당 목표 / 세션권=횟수) → 운동 목표.

### 1-2. 회원권 등록 — 2종 모델 (구현됨)

회원권은 **세션권(session)**과 **기간권(period)** 2종이며, 활용도 위협을 **형태별로 다르게 프레이밍**한다.

| 형태 | 정의 | 위협 프레이밍 | 예시 |
|---|---|---|---|
| **세션권(session)** | 횟수 차감(PT·클래스). 안 쓰면 **현금 가치가 사라짐** | **소멸** | `[만료 D-7] PT 10회권 · 4회 ₩240,000 소멸 임박` |
| **기간권(period)** | 무제한·주당 목표. 현금이 사라지진 않고 **진행이 멈춤** | **정체** | `[D-20] 헬스 3개월권 · 이번 주 정체 ₩30,000` |

- **입력 방식**: 수동 / **영수증 스캔(input_method=receipt_scan)** — OCR 자동 인식(센터·금액·기간). *등록 마찰 제거.*
- **회원권 활용도 모델(portfolio)**: 회당가치 = 원금 ÷ 목표 방문, 목표 = **원금의 75% 회수**, 25/50/75/100% 단계 마라톤, 만료 임박(D-7) 알림.

**우선순위**: 🟢 Phase 1

---

## 🏠 모듈 2 — 홈 (ROI + 효과 분석) · VOC 핵심 반영

> VOC #1 장벽("결과가 안 보임") 정면 대응. **효과 분석을 상단, ROI는 동기 연출**로.

### 2-1. 내 운동 상태 카드 (구현됨, 효과 가시성)
- 주간 7일 그래프 · 이번 주 목표 게이지(회원권 목표 개인화) · **연속 스트릭(주 단위·프리즈·최고연속)** · 최근 방문 패턴 1줄 · **지난주 대비 증감(↑↓)** · 코치 한마디(통합).

### 2-2. 회원권 상태 (구현됨)
- "회원권 N개" 요약 → 부피 줄인 미니 리스트(이름·D-day·활용도) · 빈 상태.
- ROI 히어로: 누적 회수(카운트업) · 활용도 바(단계 마커) · "오늘 체크인으로 +₩N".

### 2-3. 코치 추천 카드 (방향, ux-flow-coach-first)
- 말풍선 단독 노출 제거 → 분석 해설형 "코치 한마디". 향후 "오늘의 추천 + 실행 버튼"으로 승격.

**우선순위**: 🟢 Phase 1 (효과분석·ROI), 🟡 추천 카드 고도화 Phase 2

---

## 🗓️ 모듈 3 — 일정 (체크인 + 일정 캘린더) · VOC #2(일정 최적화)

### 3-1. 검증 체크인 (구현됨)
- 당일형/계획형. **지오펜스 100m 검증** → `verified`(회수 반영) / 자기신고 `unverified`(회수 미반영, 구분 기록).
- 경로·날씨·대중교통 안내(Edge Functions: geocode/weather/route/transit), 음성 격려.

### 3-2. 일정 캘린더 (구현됨)
- 기록 ↔ 일정 보기 전환, 월 그리드, **태그 필터(운동/식단/방문/기타)**.
- **일정 추가·편집**(schedules 테이블) · **AI 추천(운동/식단)을 일정에 추가** · **예정→완료 자동 연동**(기록 발생 시).

### 3-3. AI 최적 시간/메뉴 추천 (방향)
- 방문 패턴(요일·시간대) 기반 "토요일 9시" 추천. **우선순위**: 🟡 Phase 2 (데이터 축적 후).

# 06. Success Metrics

## 🎯 Aha Moment
첫 **검증 체크인** 완료 → 회당비용 하락/활용도 목표까지 남은 횟수 감소를 눈으로 확인하는 순간. ("생각보다 얼마 안 남았네 → 다 쓸 수 있겠다.")

## ⭐ North Star Metric — **주간 검증 체크인 수** (선행) · **회원권 활용도** (결과)

- **NSM(선행지표) = 주간 검증 체크인 수**: 매주 움직이고, 사용자의 핵심 행동(실제 운동)을 직접 측정하며, 회수(수익)에 바로 연결된다. ①가치전달 ②주단위 측정 ③수익연결을 모두 만족.
- **North-Star 결과(아웃컴) = 회원권 활용도**: NSM이 누적되며 끌어올려지는 최종 가치 지표. 후행·기간권 분모 모호로 *주단위 NSM에는 부적합*해 결과 지표로 둔다.

> **재검토 결론** — NSM은 **선행(체크인) / 결과(활용도)** 2층으로 유지한다. v0.2의 '활용도 단독 NSM'은 후행이라 주간 운영에 부적합했고, VOC도 *활용도=동기 레이어, 행동(체크인)=진짜 동력*임을 지지한다. '검증'을 NSM에 둔 이유: 자기신고 폭증으로 지표가 부풀려지는 것을 막고 회수 신뢰를 지키기 위함(검증=지오펜스).

## 🧪 베타 NSM (테스터 ≤5명 · 2~3일) — 별도 적용

위 출시용 NSM(주간 검증 체크인)은 **베타엔 부적합**하다: ①측정창이 2~3일이라 "주간"이 안 됨 ②표본 ≤5명이라 비율·코호트가 무의미 ③검증 체크인=실제 헬스장 방문(지오펜스)이라 짧은 내부 테스트에서 발생 안 할 수 있음.

> **베타 1차 지표 = "핵심 루프 활성화 완주 인원"** — 테스터가 2~3일 내 *가입 → 온보딩 → 회원권 등록 → 첫 핵심 행동 1회*까지 도달했는가. **비율이 아닌 인원수(n/5)**로 본다.

**활성화 퍼널 (인원수, n=5)**

| 단계 | 이벤트 | 베타 목표 |
|---|---|---|
| 가입 | `signup` | 5/5 |
| 온보딩 완료 | `onboarding_complete` | 5/5 |
| 회원권 등록(핵심) | `membership_added` | 5/5 |
| **첫 핵심 행동 1회+**(체크인 시도/추천 실행/일정·식단·운동 기록) | `checkin_started`·`recommend_click`·`schedule_added`·`diet_logged`·`workout_completed` | **4/5** |
| 코치 1회 사용 | `coach_open` | 3/5 |

- **베타 NSM = 첫 핵심 행동 도달 인원 ≥ 4/5** (= 앱이 아하 순간까지 데려가는가).
- **검증 체크인(지오펜스·회수)**: 정량 NSM이 아니라, **테스터 1명이 현장 1회 성공(`checkin_verified` 1건)**하면 "기능 동작 OK"로 **정성 확인**.
- **정성 우선**(n≤5): 5명 관찰·인터뷰(막힌 지점·아하 도달), 치명 버그 0건, 간단 만족도(5점).
- 베타 종료 후 사용자·기간이 확보되면 **출시용 NSM(주간 검증 체크인)으로 승격**.

# 08. Technical Architecture (현행)

```
프론트: Expo SDK 56 (React Native + RN Web), Expo Router
  ↓
Supabase: PostgreSQL + Auth + Edge Functions(Deno)
  ├─ coach / coach-chat (AI 코치·챗봇)
  ├─ food-search (식단 추천·Vision)
  ├─ workout-ai (루틴·피드백)
  └─ geocode / weather / route / transit (체크인 길안내)
  ↓
AI: Claude (ANTHROPIC_API_KEY = Supabase 시크릿, 서버 프록시)
외부: Kakao(센터 검색) · 위치(expo-location)
```

| 영역 | 기술 | 이유 |
|---|---|---|
| 앱 | Expo RN + Web | 동일 코드 웹/iOS/Android |
| 데이터·인증·서버 | Supabase | DB+Auth+Edge Functions 일원화 |
| AI | Claude (Edge Function 프록시) | 개인화·키 보안(클라 노출 X) |
| 위치/길안내 | expo-location + geocode/route/transit | 체크인 검증·경로 |
| 회원권 OCR | 영수증 스캔 | 등록 마찰 제거 |

# 09. Value Proposition

```
운동을 '기록'하게 하는 앱이 아니라,
투자한 돈·시간의 '효과와 회수'를 보이게 하고
다음 한 걸음을 제안하는 AI 페이스메이커.
```

- **여성 타깃**: "내 운동이 효과 있는지 보인다 · 내 일정에 맞는 다음 행동을 제안받는다 · 회원권이 낭비되지 않는다는 안심."
- **남성(확장 가설)**: "투자 대비 가치가 데이터로 보인다 · 나에게 맞는 운동만 한다 · 계속할 이유가 생긴다." (Phase 2 검증 대상)

### 용어 정의
| 용어 | 정의 |
|---|---|
| 세션권(session) | 횟수 차감형(PT·클래스). 안 쓰면 현금 가치 **소멸** |
| 기간권(period) | 무제한·주당 목표형. 안 쓰면 진행 **정체**(현금 소멸 아님) |

| 회원권 활용도 | 원금 대비 회수 진행도 (목표 = 원금의 75%, 단계 25/50/75/100%) |
| 검증 방문 | 지오펜스 100m 내 체크인(verified) — 회수 반영 |
| 자기신고 | GPS 미검증 방문(unverified) — 회수 미반영 |
| 유효 방문 | 방문 + (향후) 운동·식단 충족 |
| 스트릭(주) | 주 2회 충족 연속 주(프리즈 1회 보호) |
| NSM | 주간 검증 체크인 수 |
