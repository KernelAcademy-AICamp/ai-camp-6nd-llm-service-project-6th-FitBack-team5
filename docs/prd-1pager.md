# FitBack — PRD 1-Pager (prd-v0.3 기준 요약)

## 프로젝트명
**FitBack** — "Fit Back(운동 투자 가치를 되찾다) + Feedback(결과 가시화 + 다음 행동 제안)"

## 한 문장 정의 (누구를 위한 무엇)
**회원권을 끊지만 끝까지 못 쓰는 여성 직장인을 위해**, 운동에 투자한 돈·시간의 **효과와 회수를 눈에 보이게** 만들고 **AI 페이스메이커가 다음 행동을 제안**하는 지능형 피트니스 앱.

## 해결하려는 문제
회원권 결제 후 ①효과가 보이지 않아 동기가 떨어지고 ②일정이 불규칙해 미루다 ③결국 다 못 쓰고 투자금이 회수되지 않는다.
→ VOC(n=21, 구매자 14): 진짜 장벽은 "마감 깜빡함"(1명)이 아니라 **동기·결과 부재(8/14)**. "돈이 아깝다"는 인식은 있으나(10/14) 체념에 가깝다.

## 타깃 사용자
- **주타깃: 여성 직장인 25~39세** — 헬스·필라테스·요가 보유, 한 센터 정착, 환경·트레이너 중시.
  - Pain: 효과가 안 보여 동기↓ / 일정 불규칙 / 다 못 쓸 막연한 불안.
  - JTBD: "내 운동이 효과 있는지 확인하면서, 최소 노력으로 꾸준히 다니고 싶다."
- (확장 가설) 남성 직장인 30~39 — **Phase 2 검증 대상**(현재 타깃 아님).

## 솔루션 가설 (왜 작동하는가)
**결과(효과 분석)를 먼저 보여주고**, 회원권을 활용도(회수)의 '원금'으로 연출하며, **검증 체크인**으로 신뢰를 지키고, **AI 코치가 다음 한 걸음**을 제안한다.
- 근거: 회당비용/활용도를 스스로 찾는 사람은 1/21뿐이지만, **보여주면 76%가 "동기로 작동, 확인하고 싶다"**. → 활용도는 검색 기능이 아니라 **동기 연출 레이어**, 히어로는 '효과 가시성'.
- 톤: 죄책감·공포 대신 **응원·활용도(통장/재테크) 프레이밍**. 위협은 형태별 사실로(세션권=소멸 / 기간권=정체).

## 핵심 기능 3개
1. **홈 — 회원권 ROI + 효과 분석**: 회원권별 ROI 카드 캐러셀(활용도·회당가치·D-day·"오늘 출석으로 +₩N") + 주간 추이·스트릭·부위 분석. *(VOC #1 "결과 안 보임" 정면 대응)*
2. **검증 체크인 + 일정 캘린더**: 지오펜스 100m 검증(회수 반영, 검증/수동 구분) + 월 캘린더·태그 필터·AI 추천 일정 추가·예정→완료 자동연동. *(VOC #2 일정 최적화)*
3. **AI 코치 챗봇 (단일 허브)**: 데이터 해석 + 다음 행동 제안 — 운동 루틴(라이브러리 기반 RAG-lite)·식단 추천/사진 분석·일정 연계, 멀티턴·이전 대화. 전 AI는 Edge Function 서버 프록시.

## 성공 기준 (6주 베타 끝 측정 가능)
- **NSM(선행) — 주간 검증 체크인**: 활성 사용자당 **주 2회+**.
- **결과 — 회원권 활용도**: 6주간 활성 코호트 **월 +5%p** 상승 추세.
- **체크인 완주율 70%+** (시작 대비 검증 완료) — *베타 대리지표.*
- **코치 추천 실행률 50%+** (추천 → 실제 행동/방문/일정 추가).
- **효과분석 열람률 60%+** (홈 진입 중 운동상태 카드 상호작용).
- **활성화 퍼널**(가입→온보딩→회원권→핵심행동) 단계 전환율 측정·개선.
> 검증 체크인 절대수 확보가 어려운 베타 특성상, **완주율·추천 클릭율**을 대리지표로 병행.

## 기술 스택
- **프론트**: Expo SDK 56(React Native + RN Web), Expo Router · TanStack Query · Zustand · TS strict
- **백엔드**: Supabase — PostgreSQL · Auth · **Edge Functions(Deno)** (coach / coach-chat / ai-feedback / food-search / workout-ai / receipt-ocr / geocode·weather·route·transit / synth-tts)
- **AI**: Claude(Anthropic) — Edge Function **서버 프록시**(클라이언트 키 노출 X)
- **외부/네이티브**: Kakao(센터·지도) · expo-location · react-native-webview(네이티브 지도) · PostHog+analytics 뷰
- **배포**: 웹=Vercel · 안드=EAS · iOS=웹빌드 재사용
