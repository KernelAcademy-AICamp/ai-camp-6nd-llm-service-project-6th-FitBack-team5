# FitBack — 기록 캘린더 → 일정 캘린더 개선 계획

문서 버전: v0.1
작성일: 2026-06-18
근거: `ux-flow-current.md`(AS-IS) + `ux-flow-coach-first.md`(코치·추천 중심) + 개선 방향 제안(A~E)

---

## 0. 방향
계획의 **명칭(일정 캘린더)·AI 추천 일정·편집** 방향은 맞다. 다만 아래 **필수 3가지**가 빠지면 동작하지 않으므로 반드시 추가하고, **모달 유지 + 내부 "기록 ↔ 일정" 전환**을 권장한다.

---

## 1. 필수 (이게 없으면 계획이 안 돌아감)
1. **일정 저장 데이터 모델 신설** — 현재 캘린더는 과거 기록(`visits·workout_logs·meals`)만 읽음. "AI 추천을 일정에 넣고 편집"하려면 **예정 일정 테이블**이 필요.
   - `schedules(id, user_id, date, type[diet/workout/visit/custom], title, payload jsonb, source[ai/manual], status[planned/done/skipped], created_at)` + 인덱스 + RLS(owner-only).
2. **예정 ↔ 완료 상태 연결** — 일정을 **수동 완료 토글**(planned↔done)로 처리(구현됨). ※ "실제 식단/운동/체크인 기록 시 매칭 일정 자동 done"은 **오매칭 위험**이 있어 **별도 검토 항목으로 보류**.
3. **AI 추천 → "일정에 추가" 진입점** — CoachChat(운동 루틴·식단 추천)에 `[일정에 추가]` 버튼.

## 2. 보완 (개선안 C·D + 알림)
4. **검증/자기신고 구분 표시** — `visits.verify_status`로 방문 점/배지 구분. (구현됨)
5. **권장 페이스 개인화** — 고정 주2회 → 회원권 `weekly_goal` 기반. (구현됨)
6. **알림 연계** — 예정 일정 → 🔔 "오늘 예정" 노출(기존 만료 D-7과 함께). (구현됨)
7. **예정 vs 기록 시각 구분** — 완료 시 취소선/옅게 처리. (구현됨)

> ~~일정에서 바로 실행([체크인]/[운동 시작]/[식단 기록] 버튼)~~ — **불필요 기능으로 제외**(2026-06-19).

## 3. 페이지/탭 결정
- **구조 변경(하단 탭 추가/별도 라우트)은 제외** → **현재 모달 유지**.
- 모달 내부에서 **"기록 ↔ 일정" 보기 전환(세그먼트)** 정도는 구조 변경이 아니라 권장. (탭 승격은 보류)

## 4. 명칭
- "기록 캘린더" → **"일정 캘린더"**(과거 기록 + 예정 일정 모두 포함).

## 5. 진행 순서
1. **`schedules` 테이블 + CRUD 훅**(필수 토대) ← 시작
2. 캘린더에 **예정 일정 표시 + 편집(추가/수정/삭제)**
3. **CoachChat "일정에 추가"** 연결
4. **예정 ↔ 완료 상태 연결**
5. (보완) 검증 구분·알림 연계·페이스 개인화 ✅ — 단, **일정에서 바로 실행 버튼은 제외**(불필요)

## 6. 주의
- `schedules`는 공유 DB 스키마 변경(적용됨) → 팀 공지.
- 식단/운동 탭·CoachChat 일부는 팀원 파일 → 진입점 추가 시 범위 확인.

## 7. 보류(검토) 항목
- **예정→완료 자동 연동**: 실제 기록(식단/운동/체크인) 발생 시 매칭 일정 자동 `done`. **오매칭 위험**으로 보류 — 현재는 수동 완료 토글로 대체.
